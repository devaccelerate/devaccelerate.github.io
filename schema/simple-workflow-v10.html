DevAccelerate Simple Workflow

XML Specification v1.0

Schema

    <?xml version="1.0" encoding="utf-8"?>
    <da:simpleWorkflow name="workflow-name" abortOnError="true" xmlns:da="https://devaccelerate.github.io/schema/simple-workflow-v10.html">
      <da:workflowItems>
        <da:workflowItem workflowItemType="Action" name="workflow-item-name" type="workflow-item-type">
          <da:settings>
            <da:setting name="setting-name" value="setting-value" />
          </da:settings>
          <da:expectedParameters>
            <da:expectedParameter name="parameter-name" parameterType="parameter-type" required="true" />
          </da:expectedParameters>
        </da:workflowItem>   
      </da:workflowItems>
    </da:simpleWorkflow>

Elements

The simpleWorkflow Root Element

This is the root element of the XML document.

Attributes
Attribute Description Required
name The name of the workflow. Ti must be a unique value. Yes
abortOnError Determines if the workflow is to be aborted as soon as a workflow item returns an error. The default value is false. No

The workflowItems Element

The element contains one or more workflowItem elements. A workflow must have at least one workflow item.

The workflowItem Element

The element represents a single workflow item within the workflow.

Attributes
Attribute Description Required
name The name of the workflow item. It must be a unique value whtin the corresponding workflow. Yes
workflowItemType The type of the workflow item. It can be either Action which returns data or Condition which returns true or false. The default value of this attribute is Action. No
type The implementation type of the workflow item. Yes

The settings Element

The element contains one or more setting element. Settings are optional and therefore this element can be completely ignored if needed.

The setting Element

The element represents a setting for a workflow item.

Attributes
Attribute Description Required
name The name of the setting. It must be a unique value within the corresponding workflow item. Yes
value The value of the setting. No

The expectedParameters Element

The element contains one or more expectedParameter element. An expected parameter is a parameter that a workflow item needs during execution. The workflow system should throw an error if a required expeted parameter does not exist when the corresponding workflow item executes.

Expected parameters are optional and therefore this element can be completely ignored.

The expectedParameter Element

The element represents a parameter that a workflow item expects to be passed when the item is executed..

Attributes
Attribute Description Required
name The name of the expected parameter. It must be a unique value within the corresponding workflow item. Yes
parameterType The type of the expected parameter. The workflow system should validate the type of the expected parameter before the execution of the workflow item. Yes
required Determines if the expected parameter must be present when a workflow item executes. The default value is true. No

Example

    <?xml version="1.0" encoding="utf-8"?>
    <da:simpleWorkflow name="SignInWorkflow" abortOnError="true" xmlns:da="https://devaccelerate.github.io/schema/simple-workflow-v10.html">
      <da:workflowItems>
        <da:workflowItem workflowItemType="Action" name="SignIn" type="Ejyle.DevAccelerate.Workflow.Security.Authentication.DaSignInWorkflowItem, Ejyle.DevAccelerate.Workflow.Security">
          <da:settings>
            <da:setting name="httpSessionName" value="DA_USER_SESSION" />
          </da:settings>
          <da:expectedParameters>
            <da:expectedParameter name="signInInfo" parameterType="Ejyle.DevAccelerate.Workflow.Security.Authentication.DaSignInWorkflowItemInfo, Ejyle.DevAccelerate.Workflow.Security" required="true" />
          </da:expectedParameters>
        </da:workflowItem>   
      </da:workflowItems>
    </da:simpleWorkflow>
